logic3 API documentation

Summary

The logic3 package provides a simple three-valued logic system with three truth values: True, False, Nil.  These values are represented by a class named Logic3Value.  For the algerbaic properties of the structure consisting of the three truth values and the corresponding operations see the notes here.  The API provides a fluent interface to create, access and test the truth values.  All objects in the API are immutable.
Project homepage: https://github.com/hidega/logic3
Logic3
The class of the immutable base singleton object, which can be imported e.g. via   var Logic3 = require('logic3') .  This instance provides the basic constants and methods to access, test and create truth values.

Represents the True value of the three-valued logic system.

Represents the False value of the three-valued logic system.

Represents the Nil value of the three-valued logic system.

of Anything Logic3Value
Parses the input and converts it to a Logic3Value instance.  Logic3Value instances are returned as they are.   null, undefined, NaN and invalid Date objects are converted to Nil.  Empty string, zero number and false boolean values are converted to False.  Other values are converted to True.

Provides a fluent interface to test if the input is a valid Logic3Value instance. For example:

Logic3.test(someObject).ok(() => console.log('Passed')).fail(() => { throw 'Did not pass' })

Assertions
This class provides methods that support asserting conditions in tests.   The assertion methods return true on success, enabling assertions to be concatenated by  && in a single expression, such as:

var check = () => Logic3.assertions.assertTrue(varA) && Logic3.assertions.assertNotTrue(varB) && Logic3.assertions.assertEqual(varC, varD) ...

Algebraic properties of L3

The truth values and the four operations (addition, subtraction, multiplication and division) form a finite field of order 3:   F := ({ True, False, Nil }, +, -, ·, ÷).   True represents the multiplicative identity of the field, Nil represents the additive identity of the field and False represents the additive inverese of True.
